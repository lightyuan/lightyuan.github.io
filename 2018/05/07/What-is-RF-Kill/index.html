What is RF-Kill ?

RFKill 介绍

许多电脑都拥有无线发射设备，包括WI-FI, Bluetooth, 和3G设备.当这些设备不是用的时候,依然消耗功率.

RFKill是Linux内核内的一个子系统, 它提供一种接口, 通过该接口计算机系统中的无线发射设备可以被询问, 被激活, 或者被无效. 当发射装置被无效时, 它可以处于被软件再次激活的状态(a soft block), 或者处于无法再次被激活的状态(a hard block).

RFKill core 为该子系统提供了API. 内核驱动已经设计成支持RFKill使用该API向内核登记, 还包含一些方法如: 启动设备, 关闭设备. 除此以外, RFKill core 还提过用户应用可以理解的通告, 以及一些方法, 通过该方法用户应用可以查询发射装置的状态.

RFKill 接口就是 /dev/rfkill, 它包含系统中所有无线发射装置的当前状态. 每个设备都拥有它的当前RFKill状态, 该状态已注册到sysfs. 而且, RFKill 为每个RFKill-enabled设备的状态改变产生文件.

命令行工具 rfkill

rfkill是一个命令行工具, 通过它你可以查询并改变系统中RFKill-enabled设备.获得该设备需要安装rfkill包.

获取无线设备表:

$ rfkill listi

限制序号为[number]的设备:

$ rfkill block [number]

限制特定类别的设备, 如 wifi, blootooth等:

$ rfkill block wifi

限制所有设备:

$ rfkill block all
