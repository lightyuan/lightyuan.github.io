vps远程部署hexo: 基于git hook

发表于 2018-06-22

花了几乎一天的时间才搞定vps上的远程部署,原本打算用github就就可以了,不知道怎么回事,搞来搞去总是无法登录到github主页上去,直到今天我才知道,原来我在编辑hexo的部署配置文件_config.yml时,由于粗心,少打了个字母….^.^\\,导致无法创建master分支,而github主页是基于master分支的,到现在,原因显而易见了.

配置服务器远程Git仓库

添加公钥:

在本地生成秘钥,将公钥 id_rsa.pub 添加到VPS中

# vim /home/git/.ssh/authorized_keys

安装Git:

# apt-get install git

创建git用户:

添加账户:

# adduser git

修改账户登录shell:

# vim /etc/passwd

搜索用户 git 并修改为 /usr/bin/git-shell

创建Git仓库

配置 git hook

# vim /var/repo/blog.git/hooks/post-receive

添加文本:

添加执行权限:

# chomod u+x /var/repo/blog.git/hooks/post-reveive

修改相关文件目录权限和属性

本地主机配置

配置文件_config.yml中的[deploy]条目, 采用git部署方式:

# hexo # git
